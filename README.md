# LLM Wiki

**Persistent personal knowledge bases powered by LLMs — beyond traditional RAG.**

Traditional RAG forces the model to rediscover and re-synthesize knowledge on every query.  
**LLM Wiki** flips the paradigm: the LLM becomes a proactive *knowledge engineer* that reads your raw documents once and maintains a clean, interlinked, evolving Markdown knowledge base.

**Goal**: Build a living second brain that accumulates refined knowledge over time instead of starting from scratch on every question.

### Quick Start
1. Drop your documents, notes, PDFs, or articles into the `raw/` folder  
2. Paste `llm-wiki.md` into Claude, GPT, Grok, or any LLM agent  
3. Tell the agent: “You are my LLM Wiki engineer. Build and maintain the wiki from raw/”

### 2026 Novel Enhancements (from latest arXiv)
- **HiMem** & **H-MEM**: Hierarchical long-term memory for long-horizon agents (arXiv:2601.06377, 2507.22925)  
- **Agentic Memory (AgeMem)**: Unified long/short-term memory managed inside the agent policy (arXiv:2601.01885)  
- **Graph-based Agentic Memory**: Multi-graph hierarchies with automatic tracing and consolidation (arXiv:2602.05665, G-Memory)
